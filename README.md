# 工业互联网平台前端界面实现说明

本项目基于HTML5、CSS3和原生JavaScript实现了一个响应式的工业互联网平台管理界面，包含完整的侧边栏导航、多视图路由切换和设置功能。

## 1. 项目结构

- `index.html`: 页面主结构文件，包含所有视图的HTML结构。
- `style.css`: 页面样式文件，包含布局、组件样式及动画效果。
- `script.js`: 前端逻辑控制，实现路由切换和交互功能。

## 2. 功能特性

### 2.1 布局与导航
- **响应式布局**: Header + Sidebar + Main Content 结构，适配不同屏幕尺寸。
- **侧边栏导航**: 支持一级菜单和二级折叠菜单（如"系统设置"）。
- **平滑动画**: 菜单展开/折叠、视图切换均包含过渡动画。

### 2.2 前端路由 (SPA)
- 采用 **Hash模式** (`#kb`, `#settings-kb` 等) 实现无刷新页面切换。
- 支持浏览器前进/后退功能。
- 自动同步侧边栏激活状态（包括展开对应的父级菜单）。

### 2.3 视图模块
目前包含以下功能视图：
1.  **知识库 (Knowledge Base)**: 展示数据表格，支持CRUD操作（UI演示）。
2.  **系统设置**:
    -   **知识库设置**: 包含"知识库标签设置"模块，支持标签的增删改查、分页显示及关联知识库管理。
    -   **知识图谱设置**: 包含"实体类型配置"模块（支持关联文档类型）和"文档类型配置"模块，均支持增删改查操作。
        *   **数据同步**: 实体类型中的"关联文档类型"动态加载自文档类型列表；删除文档类型时，会自动清理所有关联该类型的实体配置，确保数据一致性。
    -   **备件铭牌设置**: 预留配置界面。

## 3. 技术实现细节

### 3.1 路由逻辑
在 `script.js` 中维护了一个路由映射表 `routes`：
```javascript
const routes = {
    '#kb': 'view-kb',
    '#settings-kb': 'view-settings-kb',
    // ...
};
```
通过监听 `window.onhashchange` 事件，动态切换 `.page-view` 的显示状态，并更新侧边栏的高亮样式。

### 3.2 样式扩展
- `.submenu`: 实现二级菜单的折叠/展开效果。
- `.page-view`: 控制不同业务模块的显示与隐藏，配合 `@keyframes fadeIn` 实现淡入效果。
- `.settings-form`: 统一的表单样式，用于各个设置页面。

## 4. 扩展指南

### 添加新页面
1.  在 `index.html` 的 `.page-body` 中添加新的 `<div id="view-new-page" class="page-view">...</div>`。
2.  在 `script.js` 的 `routes` 对象中添加映射：`'#new-page': 'view-new-page'`。
3.  在侧边栏添加对应的链接 `<a href="#new-page">...</a>`。

### 修改主题
继续沿用 `style.css` 中的 CSS 变量 (`:root`) 进行全局配色调整。

## 5. 兼容性
支持 Chrome, Firefox, Safari, Edge 等现代浏览器。无需构建工具，直接打开 `index.html` 即可运行。
